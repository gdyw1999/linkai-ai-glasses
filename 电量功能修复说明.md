# 电量功能修复说明

## 问题描述

用户反馈电量没有显示，虽然电量功能已经集成到HomeViewModel和ProfileViewModel中，但实际运行时电量值始终为0。

## 根本原因

在`GlassesSDKManager.kt`的`getBatteryLevel()`方法中，虽然注册了电量回调监听器，但回调函数中没有解析response数据并更新电量值。

## 修复方案

### 1. 修复电量回调解析

参考官方demo的实现（`MainActivity.kt`第551-558行），在电量回调中正确解析数据：

```kotlin
LargeDataHandler.getInstance().addBatteryCallBack("glasses_sdk_manager") { _, response ->
    // 解析电量数据
    // response.loadData[6] = 0x05 表示电量上报
    // response.loadData[7] = 当前电量
    // response.loadData[8] = 是否充电中
    if (response != null && response.loadData.size > 8) {
        val battery = response.loadData[7].toInt() and 0xFF
        val charging = response.loadData[8].toInt() and 0xFF
        updateBatteryInfo(battery, charging == 1)
        Log.d(TAG, "Battery callback: $battery%, charging=${charging == 1}")
    }
}
```

### 2. 电量数据流

```
连接成功 → queryBattery()
    ↓
addBatteryCallBack() + syncBattery()
    ↓
SDK返回电量数据 → 回调触发
    ↓
解析response.loadData[7] (电量) 和 loadData[8] (充电状态)
    ↓
updateBatteryInfo() → 更新 _batteryLevel Flow
    ↓
HomeViewModel/ProfileViewModel 监听 batteryLevel Flow
    ↓
UI自动更新显示电量
```

### 3. 已集成的电量显示位置

#### HomeScreen (首页)
- 顶部状态栏：显示设备名称和电量
- 连接状态卡片：显示电量百分比和图标

#### ProfileScreen (我的页面)
- 设备信息卡片：显示设备名称和电量
- 格式：`电量: XX%`
- 只在已连接且电量>0时显示

## 注意事项

1. **电量数据延迟**：SDK返回电量数据可能需要1-2秒，UI应该有加载状态提示
2. **断开连接时清空电量**：断开连接后应该将电量重置为0
3. **错误处理**：如果电量查询失败，应该有友好的错误提示
4. **数据有效性**：电量值应该在0-100范围内，需要做边界检查
