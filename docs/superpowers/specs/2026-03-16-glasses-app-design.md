## 一、项目概述

### 1.1 项目定位

青橙AI眼镜App是一款基于Android平台开发的智能眼镜控制应用，集成了：

- **设备控制**: 通过青橙SDK实现蓝牙连接、拍照、录像、录音等功能
- **AI对话**: 集成LinkAI平台，实现语音识别、对话生成、语音合成
- **媒体管理**: WiFi同步眼镜上的媒体文件，支持浏览和删除

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 便捷控制 | 一键触发眼镜拍照/录像/录音 |
| AI助手 | 语音对话，智能问答 |
| 媒体同步 | 自动同步眼镜拍摄内容到手机 |

## 四、核心功能模块设计

### 4.1 蓝牙连接模块

| 功能 | API | 说明 |
|------|-----|------|
| 扫描设备 | `BleScannerHelper.scanDevice()` | 扫描眼镜 |
| 连接设备 | `BleOperateManager.connectDirectly()` | 连接指定设备 |
| 断开连接 | `BleOperateManager.unBindDevice()` | 断开连接 |
| 连接状态 | `GlassesDeviceNotifyListener` | 监听连接/断开 |
| 电量同步 | `LargeDataHandler.syncBattery()` | 获取电量 |
| 版本信息 | `LargeDataHandler.syncDeviceInfo()` | 获取版本 |

### 4.2 AI对话模块

| 步骤 | LinkAI API | 说明 |
|------|------------|------|
| 1. 语音识别 | `POST /v1/audio/transcriptions` | 音频→文字 |
| 2. 对话生成 | `POST /v1/chat/memory/completions` | 文字→文字(带记忆) |
| 3. 语音合成 | `POST /v1/audio/speech` | 文字→音频 |

**触发源**:
- App按钮：`View.onTouchListener` 长按事件
- 眼镜按键：SDK事件监听 `GlassesDeviceNotifyListener`
- 眼镜唤醒词：待V1.1实现（需SDK支持唤醒词回调）

### 4.3 会话管理模块

**存储**: SQLite / Room

**表结构**:

Table: conversations
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| title | String | 会话标题 |
| created_at | Long | 创建时间 |
| updated_at | Long | 更新时间 |

Table: messages
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| conversation_id | String | 外键 |
| role | String | user/assistant |
| content | String | 文字内容 |
| audio_path | String | 本地音频路径（可选） |
| created_at | Long | 时间 |

**索引设计**:
```sql
-- 消息表按会话ID索引，优化查询性能
CREATE INDEX idx_messages_conversation_id ON messages(conversation_id);

-- 会话表按更新时间索引，支持按时间排序
CREATE INDEX idx_conversations_updated ON conversations(updated_at DESC);
```

```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />

<!-- WiFi权限 -->
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
<uses-permission android:name="android.permission.CHANGE_WIFI_STATE" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

<!-- 位置权限 (蓝牙扫描必需) -->
<!-- Android 13+ -->
<uses-permission android:name="android.permission.NEARBY_WIFI_DEVICES" />
<!-- Android 12 及以下 -->
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
```

**运行时权限处理**:

所有危险权限（蓝牙扫描、位置、存储等）需要在运行时动态请求。使用XXPermissions库统一处理：

```kotlin
// 请求所有必要权限
XXPermissions.with(this)
    .permission(Permission.BLUETOOTH_SCAN)
    .permission(Permission.ACCESS_FINE_LOCATION)
    .permission(Permission.READ_EXTERNAL_STORAGE)
    .request { _, allGranted ->
        if (allGranted) {
            // 权限已授予，开始扫描设备
        } else {
            // 显示权限说明，引导用户开启
        }
    }
```

## 九、错误处理策略

### 9.1 蓝牙连接错误

| 错误场景 | 处理策略 |
|----------|----------|
| 扫描超时 | 提示"未发现设备"，提供重新扫描按钮 |
| 连接失败 | 提示错误信息，3秒后自动重连，最多重试3次 |
| 连接断开 | 显示断开状态，引导用户重新连接 |
| 权限拒绝 | 跳转设置页面，引导开启蓝牙和位置权限 |

### 9.2 网络请求错误

| 错误场景 | 处理策略 |
|----------|----------|
| 网络不可用 | 提示"网络不可用"，缓存请求待网络恢复 |
| 请求超时 | 提示"请求超时"，提供重试按钮 |
| API鉴权失败 | 提示"认证失败"，引导重新登录 |
| 响应格式错误 | 记录日志，显示"服务异常" |

### 9.3 AI对话错误

| 错误场景 | 处理策略 |
|----------|----------|
| 录音失败 | 提示"录音失败，请检查麦克风权限" |
| 语音识别失败 | 显示"识别失败，请重试" |
| 对话生成失败 | 显示"AI助手暂时无法回答" |
| 语音合成失败 | 仅显示文字，隐藏语音播放按钮 |

### 9.4 媒体同步错误

| 错误场景 | 处理策略 |
|----------|----------|
| WiFi连接失败 | 提示"请确保眼镜和手机在同一WiFi" |
| 下载中断 | 记录已下载文件，支持断点续传 |
| 存储空间不足 | 提示"存储空间不足，请清理后重试" |
| 文件损坏 | 跳过该文件，继续下载其他文件 |
