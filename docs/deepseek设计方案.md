### 一、项目背景与目标

#### 1.1 背景
- **硬件**：青橙无线眼镜，通过官方 Android SDK（AAR）进行控制。
- **AI能力**：基于 OpenClaw（小龙虾）开源项目，公司进行专项优化，并通过 LinkAI 平台对外提供服务。
- **用户需求**：开发一款 AR 眼镜上的个人助理 App，实现语音对话交互，支持文字显示与语音播报同步。

#### 1.2 目标
- **核心功能**：
  - 设备连接与管理（蓝牙扫描、连接、状态显示）。
  - 双唤醒方式：物理按键唤醒 + 设备预置固定唤醒词唤醒（如“嘿塞恩”）。
  - 语音对话：录音 → 语音识别（STT）→ LinkAI 对话（流式）→ 语音合成（TTS）→ 文字显示 + 音频播放。
  - 媒体管理：同步眼镜拍摄的照片、视频、录音，分类查看与管理。
  - 设置功能：设备信息、佩戴检测、录制设置、固件升级、语音助手设置（唤醒开关、音色选择）等。
- **非功能目标**：
  - 优先保证对话的快速响应与流式体验。
  - UI 设计参考官方 App 截图，简洁易用。
  - 兼容华为 HarmonyOS（基于 Android 兼容层）。

### 三、模块详细设计

#### 3.1 唤醒模块
- **按键唤醒**：
  - 通过青橙 SDK 提供的按键监听接口（需查阅 SDK 文档），注册回调。
  - 短按（单击）触发唤醒，长按可预留其他功能（如拍照）。
  - 防抖处理：200ms 内只响应一次。
- **语音唤醒**：
  - 使用眼镜出厂预置的固定唤醒词（如“嘿塞恩”）。
  - SDK 应提供唤醒事件回调，App 注册监听。
  - 在“语音助手设置”界面提供开关，调用 SDK 的启用/禁用接口（若支持），并提示用户开启可能增加耗电。
- **功耗提示**：参考官方截图，在设置中添加说明：“开启后设备需时刻工作，这会让电量消耗速度加快，建议不使用时关闭。”

#### 3.2 录音管理
- **启动录音**：唤醒成功后，调用 `glassesControl.startRecording()` 开始眼镜端录音。
- **结束录音**：
  - 按键模式：再次单击按键结束录音。
  - 超时模式：设置静音检测超时（如 5 秒无声音）自动结束，或最长录音时间（如 30 秒）。
- **下载音频**：录音结束后，通过 `GlassesControl.importAlbum()` 获取最新录音文件。需确认 SDK 返回的文件路径或流。
  - 已知录音格式为 **WAV**，可直接使用。
  - 将文件保存到手机临时目录（如 `cacheDir/recordings/`）。

#### 3.3 语音识别（STT）
- **接口**：`POST https://api.link-ai.tech/v1/audio/transcriptions`
- **请求头**：`Authorization: Bearer YOUR_API_KEY`，`Content-Type: multipart/form-data`
- **请求体**：`file` 字段携带 WAV 文件。
- **响应**：`{"text": "识别出的文本"}`
- **错误处理**：捕获异常，重试机制（如 3 次），失败时提示用户。

#### 3.4 对话接口（流式）
- **接口**：`POST https://api.link-ai.tech/v1/chat/completions`
- **请求头**：`Authorization: Bearer YOUR_API_KEY`，`Content-Type: application/json`
- **请求体示例**：
  ```json
  {
    "app_code": "你的应用code",
    "messages": [
      {"role": "system", "content": "你是一个贴心的个人助理，用中文回答，简洁友好。"},
      {"role": "user", "content": "查询一下十堰的天气"}
    ],
    "stream": true,
    "stream_options": {"include_usage": true}
  }
  ```
- **记忆维护**：
  - 在 App 内维护一个 `MutableList<Message>`，存储最近的 N 轮对话（如 10 轮）。
  - 每次请求时将历史列表传入 `messages`。
  - 首次启动时，可从本地配置或应用中获取系统提示词。
- **流式解析**：
  - 使用 OkHttp 或 Retrofit 的流式请求，逐行读取以 `data: ` 开头的行。
  - 解析 JSON，提取 `delta.content`，累积到文本缓冲区。
  - 当遇到 `[DONE]` 时结束。

#### 3.5 语音合成（TTS）与播报
- **攒句策略**：
  - 缓冲区累积流式返回的文本片段。
  - 遇到句子结束符（。！？）或缓冲区长度超过 20 字时，切分为一个完整句子。
  - 将句子送入 TTS 队列。
- **TTS 调用**：
  - **接口**：`POST https://api.link-ai.tech/v1/audio/speech`
  - **请求头**：`Authorization: Bearer YOUR_API_KEY`，`Content-Type: application/json`
  - **请求体**：
    ```json
    {
      "input": "要合成的句子",
      "app_code": "你的应用code"   // 使用应用中配置的音色
    }
    ```
  - **响应**：二进制 MP3 数据。
- **播放管理**：
  - 将 MP3 数据写入临时文件（如 `cacheDir/tts/sentence_001.mp3`）。
  - 使用 `MediaPlayer` 顺序播放队列中的文件。
  - 播放完成后删除临时文件。
- **打断机制**：
  - 当用户再次唤醒并开始新录音时，立即停止当前播放，清空 TTS 队列和缓冲区。
  - 调用 `mediaPlayer.stop()` 和 `mediaPlayer.release()`。

#### 3.6 媒体管理（相册）
- **数据获取**：调用 `GlassesControl.importAlbum()` 获取媒体列表（图片、视频、录音）。需确认返回的数据结构（如 JSON 数组）。
- **分类展示**：参考截图，按“全部/图片/视频/录音”分类，支持按时间分组（今天/昨天/更早）。
- **操作功能**：预览、播放、删除、收藏（收藏状态可本地存储或同步到云端）。

#### 3.7 设置界面
- **语音助手设置**（参考 `语音01.jpg`、`语音02.jpg`）：
  - 语音唤醒开关（调用 SDK 接口控制硬件监听）。
  - 音色选择：从 LinkAI 音色列表获取选项，或直接使用应用默认。
  - AI 图像清晰度（预留）。
  - 操作引导：展示两种唤醒方式说明。
- **我的眼镜**（参考 `我的眼睛01.jpg`）：
  - 设备信息（名称、MAC 地址、固件版本、电量、连接状态）。
  - 佩戴检测开关。
  - 录制设置（分辨率、视频质量等，需 SDK 支持）。
  - 固件升级（调用 SDK 的 OTA 接口）。
  - 重启、恢复出厂设置、解除配对。

#### 3.8 主界面
- 参考 `主界面01.jpg`、`主界面02.jpg`：
  - 顶部显示设备连接状态和电量。
  - 快捷功能卡片：拍照、录像、录音（直接调用 SDK 对应功能）。
  - 智能图像识别入口（预留）。
  - 功能设置入口：跳转至“学习拍照和录像”、“触控板手势”、“语音助手”、“AI翻译”等。
  - 底部导航：首页、AI 对话、相册、我的。

### 四、LinkAI 接口配置说明

| 接口 | 用途 | 请求方式 | 关键参数 | 响应格式 |
|------|------|----------|----------|----------|
| 语音识别 | WAV 转文字 | POST multipart | `file` (WAV) | JSON `{"text": "..."}` |
| 对话 | 流式对话 | POST JSON | `app_code`, `messages`, `stream=true` | SSE 流式 JSON |
| 语音合成 | 文本转 MP3 | POST JSON | `input`, `app_code` (或 `voice`) | 二进制 MP3 |

- **认证**：所有接口均需在 Header 中携带 `Authorization: Bearer YOUR_API_KEY`。
- **应用配置**：在 LinkAI 平台创建“应用”，绑定所需的模型、知识库、插件，并在应用中配置默认音色。调用对话接口时传入 `app_code`，即可使用该应用的设定。
- **音色列表**：可在平台文档或管理后台查看所有可用 `voice` 编码，如 `BV700_V2_streaming`。

### 五、数据流时序图（一次完整对话）

```
用户唤醒 (按键/语音) 
    → App: 启动眼镜录音 (glassesControl.startRecording())
用户说话结束 (再次按键/超时)
    → App: 停止录音 (glassesControl.stopRecording())
    → App: 从眼镜下载最新 WAV 文件 (GlassesControl.importAlbum())
    → App: 上传 WAV 至 LinkAI STT (POST /v1/audio/transcriptions)
    ← LinkAI: 返回文本 T
    → App: 准备消息历史，调用 LinkAI 对话接口 (POST /v1/chat/completions, stream=true)
    ← LinkAI: 流式返回 chunks (SSE)
        循环：
            App: 解析 chunk，累积文本到缓冲区
            if (攒成完整句子) {
                句子 S = 缓冲区内容
                → App: 调用 LinkAI TTS (POST /v1/audio/speech) 合成 S
                ← LinkAI: 返回 MP3 二进制数据
                App: 将 MP3 加入播放队列
                App: 在 UI 上显示句子 S
            }
        直至流式结束
    → App: 顺序播放队列中的 MP3 文件
```

### 六、开发注意事项

#### 6.1 权限管理
| 权限 | 用途 | 备注 |
|------|------|------|
| `BLUETOOTH_SCAN` (Android 12+) | 蓝牙扫描 | 需动态申请 |
| `BLUETOOTH_CONNECT` (Android 12+) | 蓝牙连接 | 需动态申请 |
| `ACCESS_FINE_LOCATION` / `ACCESS_COARSE_LOCATION` | 蓝牙扫描必需 | Android 12 以下需位置权限 |
| `READ_EXTERNAL_STORAGE` / `WRITE_EXTERNAL_STORAGE` | 读写录音文件 | 若 targetSdk>=30，可使用分区存储 |
| `INTERNET` | 网络请求 | 正常声明 |
| `RECORD_AUDIO` | 手机端录音（备用） | 若使用手机麦克风，但主要用眼镜录音，可酌情申请 |

#### 6.2 后台保活
- 语音唤醒监听可能需要在后台运行，建议使用前台服务，并显示一个常驻通知。
- 在华为/HarmonyOS 设备上，引导用户将 App 加入“受保护应用”列表，防止被系统杀死。

#### 6.3 文件清理
- 录音文件（WAV）和 TTS 临时文件（MP3）应及时删除，避免占用存储空间。可使用 `File.delete()` 或 `Context.deleteFile()`。

#### 6.4 错误处理
- 网络请求失败时，应显示友好提示，并提供重试按钮。
- API 返回错误码时（如 402 应用不存在、406 余额不足），根据具体信息提示用户。
- 录音下载失败、TTS 合成失败等情况，应有降级方案（如仅显示文字，使用原生 TTS 备用）。

#### 6.5 流式对话的 UI 反馈
- 在等待 AI 回复时，显示“正在输入...”或动画。
- 逐句显示文字，可配合语音播报高亮当前句子。

### 八、后续优化方向

1. **边录边传**：将录音分片上传 STT，减少等待时间。
2. **流式 TTS**：若 LinkAI 后续提供流式 TTS，可实现更平滑的播报。
3. **多模态交互**：在对话接口中附加图片 URL，实现“拍物问答”。
4. **离线备用**：在网络不佳时使用 Android 原生 STT 和 TTS，并支持简单指令（如“拍照”、“录像”）。
5. **个性化音色**：允许用户录制自己的声音训练定制音色（需平台支持）。
